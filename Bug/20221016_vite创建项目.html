    # 使用 yarn 管理工具创建 vite 项目

    # 使用yarn create vite react-goodsCar 时出现错误

    今天在学习 react 时, 在使用 yarn create 创建 react 项目时, 出现报错

    C:\Users\Administrator>yarn create vite react-demo
yarn create v1.22.5
warning package.json: No license field
[1/4] Resolving packages...
[2/4] Fetching packages...
[3/4] Linking dependencies...
[4/4] Building fresh packages...

success Installed "create-vite@3.1.0" with binaries:
      - create-vite
      - cva
文件名、目录名或卷标语法不正确。
error Command failed.
Exit code: 1
Command: C:\Users\Administrator\AppData\Local\Yarn\bin\create-vite
Arguments: react-demo
Directory: C:\Users\Administrator
Output:

info Visit https://yarnpkg.com/en/docs/cli/create for documentation about this command.

    ​ 前段时间使用yarn create vite创建时没有遇到这种问题, 可能是版本更新导致。同时我也在想, 是不是和我之前清理 c 盘 yarn 的缓存时, 将 yarn 的包缓存地址放到了 d 盘导致的, 后来又尝试了使用 npm cnpm 都没用, 一样的错误。也在网上找了大半天, 大佬都是说是 yarn 的安装地址和包的缓存地址不一致导致的, 大佬的处理方式做了, 也没用。

    ​ 后面自己尝试了一些命令, 最后试出来, 使用 yarn create-vite + 项目名称或者使用 yarn cva + 项目名称 就可以啦

    # 使用 yarn create-vite 出现错误

    过了几天, 我又去用 yarn create-vite 创建项目时, 又报错了

    C:\Users\Administrator>yarn create-vite react-demo
yarn run v1.22.5
warning package.json: No license field
error Command "create-vite" not found.
info Visit https://yarnpkg.com/en/docs/cli/run for documentation about this command.

    难道是命令又更新了, 我又去网上找答案, 结果一无所获, 然后又尝试了新的命令

    create-vite它又可以了, 不晓得后面还会不会更新, 现在都不需要 yarn 了

    希望对你有用!
